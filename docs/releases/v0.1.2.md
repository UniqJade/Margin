# Margin v0.1.2

Margin v0.1.2 makes its two passage-reading modes serve visibly different
purposes while keeping the same translation request and cached result.

## Highlights

- Natural Translation now leads with the complete Chinese passage.
- The English original is available from a collapsed disclosure rather than
  competing with the translation for attention.
- Bilingual View presents ordered English–Chinese blocks with sentence numbers
  or sentence ranges, clearer spacing, and stronger orange navigation markers.
- Copy and Speak follow the active reading mode: Chinese only in Natural
  Translation, or the visible English–Chinese pairs in Bilingual View.
- Results with fewer than two alignment blocks stay in Natural Translation
  without showing a technical warning.
- The standard macOS app menu now provides **Settings…** with `⌘,`; the
  menu-bar command opens that same singleton Settings window.
- Follow System now clears Margin's app-wide appearance override, so title bars,
  settings content, saved items, and the lookup panel switch together.
- A deterministic local typography pass presents Chinese quotation marks,
  full-width punctuation, and CJK spacing consistently, including old cached
  results, Copy, Speak, word examples, notes, and saved summaries.
- Chinese passage text keeps its Songti reading face on macOS and applies local
  spacing only after Chinese commas, so they no longer resemble an inserted
  space while retaining the correct full-width `，` character. This changes
  only rendering; copied text and provider output remain unchanged.
- A compact version of Margin's existing page icon gives the passage header a
  clearer identity without reducing reading space.
- Apple Books source and copyright footers are removed before selected text is
  sent for translation. The same conservative cleanup applies to returned and
  previously cached passage results, so old polluted cache entries can be
  reused without another provider request.

## Compatibility

This release does not change structured provider output, cache or saved-data
formats, Keychain storage, permission behavior, or the 280–620 pt Mac panel
limits. Recognized terminal Apple Books attribution metadata is excluded from
translation requests and passage results. The typography pass and attribution
cleanup do not consume an additional provider request.

## Install

This is a source-only release. It intentionally contains no `.app`, DMG, or
other binary asset. Build and sign your personal copy by following
[`docs/building.md`](https://github.com/UniqJade/Margin/blob/main/docs/building.md).

## Important limitations

- English → Simplified Chinese only.
- Network and cloud provider required; no offline dictionary or local model.
- Direct client-side BYOK is appropriate only for a locally built personal
  prototype.
- AI-generated definitions and translations can be wrong and do not cite a
  licensed authoritative dictionary.
- The verified Apple Books shortcut requires user-approved Accessibility
  access on Mac.

## Privacy reminder

Do not paste API keys, private book excerpts, provider response bodies, signing
files, or populated `Local.xcconfig` values into public issues. See
[`SECURITY.md`](https://github.com/UniqJade/Margin/blob/main/SECURITY.md) before
reporting a security problem.
